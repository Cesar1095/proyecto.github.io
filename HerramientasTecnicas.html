Herramientas y técnicas para incrementar la calidad en el código del proyecto

En esta presentación son presentan varios puntos importantes que debemos de tomar en cuenta a la hora de desarrollar algún proyecto.

  • Mantener a los desarrolladores alimentados
  • Mantener a los desarrolladores relajados
  • El software perfecto no existe, siempre habrá errores, pero…
  • Afortunadamente exite el software de gran calidad

Para un software de calidad nos presentan los siguientes puntos:

  1. Debe cumplir su propósito
  2. Informar adecuadamente cuando ocurre un problema
  3. Tiene pruebas
  4. Se entiende su intención con tan solo leer el código
  5. Se sigue un estilo de código desde el inicio
  6. Utiliza un flujo de colaboración adecuado
  7. Implementa correctamente los patrones de diseño
  8. Se puede utilizar intuitivamente.

Para el manejo e información de errores, nos da dos opciones:

  • Si el error es culpa del usuario, es importante hacerle saber la causa
  • Si el error no fue culpa del usuario, se vale no entrar en detalles técnicos.

También nos habla sobre los diferentes tipos de pruebas que son:

  • Pruebas unitarias: garantiza que las clases de un programa haga lo que deben hacer
  • Pruebas integrales: verifican que los componentes de la aplicación funcionan correctamente actuando en conjunto
  • Pruebas funcionales: en esta prueba el usuario interactúa con nuestro programa, el objetivo es comprobar que el software que se ha creado cumple con la función para la que se había pensado.
